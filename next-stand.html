The Next Stand

The idea that Web standards should be open and efforts to make them so have served us well. Over time, however, the ways in which people build the Web have evolved. The ways in which we cooperate have improved. Much that was experimental in open source projects a decade or two ago, and even more that we never foresaw back then, now forms the bedrock of Web development.

The manner in which we create Web standards today lags behind the Web community’s common practices. In 2012, a small group of standards organisations came together to affirm the principles of openness known as the “Open Stand”. Fine principles, but no longer sufficient. In order to deliver on the Web’s promise, we need to operate so as to grant the full Web community stewardship over the platform. The Web has disrupted many aspects of technology. It is time to have it disrupt the way in which we produce Web standards. That is the Next Stand.

Web technology is a commons. We need to manage it as such.

There is much to do but examples such as the RICG or the DOM show that it is within reach and tell us of the pitfalls to avoid. This document is not intended to be final; rather, in keeping with the principles it upholds, it is open for all to edit.
